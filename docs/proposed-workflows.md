# Proposed Workflows

These workflows show how companies can apply the skill catalog to real operating rhythms. They are starting points, not mandatory process designs.

## Executive Operating Rhythm

Purpose: convert scattered leadership input into decisions, actions, and executive-ready reporting.

Recommended skills:

- `meeting-intelligence`
- `weekly-executive-report`
- `decision-memo`
- `project-status-brief`

Flow:

1. Run `meeting-intelligence` after leadership meetings.
2. Convert unresolved decisions into `decision-memo` drafts.
3. Roll project updates into `project-status-brief`.
4. Use `weekly-executive-report` to synthesize the operating narrative.
5. Review owners, dates, escalations, and decision asks before distribution.

Success indicators:

- fewer ambiguous follow-ups;
- faster decision documentation;
- cleaner escalation paths;
- reports that leaders can scan quickly.

## Chief AI Officer Pulse

Purpose: give AI leadership a practical operating view of adoption, portfolio health, governance risks, and executive decisions.

Recommended skills:

- `cao-operating-pulse`
- `automation-opportunity-map`
- `policy-impact-analysis`
- `research-synthesis-brief`

Flow:

1. Collect initiative updates, usage signals, incidents, policy notes, and vendor claims.
2. Run `cao-operating-pulse` to separate signal from theatre.
3. Use `automation-opportunity-map` for new candidate workflows.
4. Use `policy-impact-analysis` when governance or regulatory changes affect rollout.
5. Convert external research into evidence using `research-synthesis-brief`.

Success indicators:

- AI portfolio decisions have owners and deadlines;
- shadow AI risks are named without panic;
- pilots are separated from scaled workflows;
- board narratives are clear and evidence-based.

## Engineering Release Readiness

Purpose: reduce time spent interpreting noisy delivery signals.

Recommended skills:

- `ci-failure-triage`
- `pr-review-brief`
- `release-notes-generator`

Flow:

1. Run the CI signal extractor on long logs when available.
2. Use `ci-failure-triage` to produce root-cause hypotheses and fix paths.
3. Use `pr-review-brief` to focus review on risk, behavior, and missing tests.
4. Use `release-notes-generator` to translate merged changes into audience-ready release communication.

Success indicators:

- fewer false root-cause assumptions;
- clearer owner routing;
- release notes reflect user impact;
- review comments are actionable rather than stylistic noise.

## Revenue Operations Hygiene

Purpose: improve pipeline quality and forecast confidence without inventing CRM data.

Recommended skills:

- `crm-hygiene-auditor`
- `account-research-brief`
- `proposal-drafting-assistant`

Flow:

1. Export CRM records to CSV and run the hygiene audit helper.
2. Use `crm-hygiene-auditor` to prioritize cleanup actions.
3. Use `account-research-brief` for key accounts with active opportunities.
4. Use `proposal-drafting-assistant` after needs, constraints, and review requirements are clear.

Success indicators:

- missing owners and stale next steps are visible;
- forecast risks are evidence-based;
- account research is tied to action;
- proposals preserve assumptions and review needs.

## Knowledge Capture And Reuse

Purpose: turn ephemeral expertise into maintainable knowledge-base articles.

Recommended skills:

- `knowledge-base-capture`
- `research-synthesis-brief`
- `policy-impact-analysis`

Flow:

1. Capture meeting notes, support threads, incident learnings, or process drafts.
2. Use `knowledge-base-capture` to create a reusable article.
3. Run the metadata checker before publishing.
4. Use `research-synthesis-brief` for evidence-heavy topics.
5. Use `policy-impact-analysis` when the article depends on policy or compliance changes.

Success indicators:

- articles have audience, owner, summary, and freshness metadata;
- unresolved assumptions are not published as facts;
- related links and source provenance are preserved;
- review questions are explicit before publication.

## Automation Intake

Purpose: evaluate automation opportunities before building integrations or scripts.

Recommended skills:

- `automation-opportunity-map`
- `decision-memo`
- `project-status-brief`

Flow:

1. Collect workflow notes, systems involved, pain points, frequency, risks, and constraints.
2. Use `automation-opportunity-map` to classify opportunities by skill, script, integration, or process redesign.
3. Use `decision-memo` for build/buy/defer decisions.
4. Use `project-status-brief` once implementation starts.

Success indicators:

- quick wins are narrow and measurable;
- unverified API or data assumptions remain marked as unknown;
- rejected ideas stay out of the roadmap;
- first experiments have clear success measures.
