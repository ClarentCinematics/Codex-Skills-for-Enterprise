# Support

Use GitHub issues for repository support.

## Ask For Help

Open a documentation issue when:

- installation instructions are unclear;
- a workflow manual needs more detail;
- an example does not match a real enterprise use case;
- a link, command, or validation instruction appears outdated.

Open a skill request when:

- a recurring enterprise workflow is not covered;
- an existing skill needs a clearer output standard;
- a skill needs a deterministic helper script;
- a pack should include or exclude a skill.

## Before Opening An Issue

Please check:

- [User Manual](docs/user-manual.md)
- [Proposed Workflows](docs/proposed-workflows.md)
- [Skill Quality Standard](docs/skill-quality-standard.md)
- [Curation Policy](docs/curation-policy.md)

Include the skill name, command, source artifact type, expected output, and actual behavior when relevant.

## Scope

This repository provides skill instructions, references, scripts, validation, and adoption guidance. It does not provide managed deployment, customer-specific compliance approval, or legal review.
