# Knowledge Base Patterns

## Article Types

| Type | Use when | Core sections |
| --- | --- | --- |
| SOP | repeatable process | purpose, prerequisites, steps, owner, exceptions |
| Troubleshooting | known failure or support issue | symptoms, cause, fix, escalation |
| Incident learning | post-incident knowledge | context, impact, root cause, prevention |
| Onboarding | new user/team enablement | overview, first steps, links, support |
| Reference | stable policy/process detail | definitions, rules, examples, owner |

## Freshness Metadata

Include when possible:

- owner;
- last reviewed date;
- next review date;
- source artifacts;
- confidence level;
- publishing status.

## Publishing Readiness

- **Ready**: facts are complete, owner is clear, no sensitive leakage.
- **Needs review**: assumptions, missing owner, or incomplete steps.
- **Do not publish**: sensitive, unverified, contradictory, or legally risky content.
