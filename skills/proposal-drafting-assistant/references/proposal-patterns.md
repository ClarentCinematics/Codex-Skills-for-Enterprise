# Proposal Patterns

## Proposal Modes

| Mode | Use when | Emphasis |
| --- | --- | --- |
| Executive proposal | leadership approval is needed | business outcomes, risk, confidence |
| Pilot proposal | scope is limited and exploratory | learning goals, timeline, success criteria |
| SOW-style draft | delivery detail is needed | scope, assumptions, exclusions, dependencies |
| Renewal/expansion | existing customer context exists | outcomes delivered, new value, adoption plan |

## Required Guardrails

- Mark unconfirmed pricing as `Not provided`.
- Mark legal terms as requiring review.
- Mark customer claims that need evidence.
- Avoid guaranteed ROI unless explicitly provided and approved.

## Outcome Statement Pattern

```text
We propose <solution> to help <customer/team> achieve <business outcome> by <approach>, with success measured by <metric or observable result>.
```
